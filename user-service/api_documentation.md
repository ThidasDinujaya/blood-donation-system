# REST API and SQL Queries Documentation

Here is a breakdown of the REST API endpoints and the underlying SQL queries generated by Spring Data JPA.

---

### Authentication API

| Method | Path                      | Description                         | SQL Query(s)                                                              |
| :----- | :------------------------ | :---------------------------------- | :------------------------------------------------------------------------ |
| **POST**   | `/api/users`              | Registers a new user.               | 1. `SELECT ... FROM user WHERE email = ?`<br>2. `INSERT INTO user (...) VALUES (...)` |
| **POST**   | `/api/sessions`           | Creates a new session (login).      | `SELECT ... FROM user WHERE email = ?`                                    |
| **DELETE** | `/api/sessions/current`   | Deletes the current session (logout). | *(No database query)*                                                     |
| **GET**    | `/api/emails`             | Checks if an email is registered.   | `SELECT ... FROM user WHERE email = ?`                                    |

---

### User API

| Method | Path                   | Description                                  | SQL Query(s)                                                                                             |
| :----- | :--------------------- | :------------------------------------------- | :------------------------------------------------------------------------------------------------------- |
| **GET**    | `/api/users`           | Finds users with filters and pagination.     | `SELECT ... FROM user WHERE (city = ? AND blood_group = ? AND available_to_donate = ?) LIMIT ? OFFSET ?` |
| **GET**    | `/api/users/{id}`      | Retrieves a single user by their ID.         | `SELECT ... FROM user WHERE id = ?`                                                                      |
| **PUT**    | `/api/users/{id}`      | Updates a user's profile information.        | 1. `SELECT ... FROM user WHERE id = ?`<br>2. `UPDATE user SET ... WHERE id = ?`                           |
| **DELETE** | `/api/users/{id}`      | Deletes a user by their ID.                  | 1. `SELECT count(*) FROM user WHERE id = ?`<br>2. `DELETE FROM user WHERE id = ?`                         |
| **GET**    | `/api/users/me`        | Retrieves the profile of the current user.   | `SELECT ... FROM user WHERE email = ?`                                                                   |
| **PUT**    | `/api/users/me/password` | Updates the current user's password.         | 1. `SELECT ... FROM user WHERE email = ?`<br>2. `UPDATE user SET password = ? WHERE id = ?`               |

---

### Statistics API

| Method | Path                               | Description                             | SQL Query                                               |
| :----- | :--------------------------------- | :-------------------------------------- | :------------------------------------------------------ |
| **GET**    | `/api/blood-groups/stats/user-counts` | Returns a count of users for each blood group. | `SELECT u.blood_group, COUNT(u.id) FROM user u GROUP BY u.blood_group` |

---

**Note:** The exact SQL syntax (e.g., column names, table name) may vary slightly depending on your `User` entity definition and your JPA naming strategy. The queries shown are representative of what Spring Data JPA generates from your repository methods.
